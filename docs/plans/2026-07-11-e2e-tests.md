## Ground Rules for Implementing Agents

1. **Framework:** Playwright (TypeScript). Install: `npm install -D @playwright/test`
2. **All tests run against the live Vercel URL.** Set `BASE_URL` in `playwright.config.ts` to the production URL.
3. **Auth:** Tests use a dedicated test Google account (`TEST_GOOGLE_EMAIL` / `TEST_GOOGLE_PASSWORD` env vars). This account must have real financial emails in Gmail — otherwise sync tests use demo data.
4. **No mocking.** These are live E2E tests. Real Gmail API, real Gemini, real DB. Some tests have a "demo data" variant for running without a real Google account.
5. **Test isolation:** Each test suite that writes data (sync, categories, assets) must clean up via `DELETE /api/user/data` in `afterAll`, or use a dedicated test user per suite.
6. **Flake prevention:** All sync-related tests use generous timeouts (300 seconds for scan, 600 seconds for full sync). Polling tests retry every 15 seconds.
7. **Run order:** Execute suites in the order listed — auth must pass before all others; onboarding before sync; sync before transactions assertions.
8. **File location:** All test files go in `e2e/` directory at project root.
9. **Test command:** Add `"test:e2e": "playwright test"` to `package.json` scripts.
10. **Before writing any code:** Run `cat node_modules/next/dist/docs/01-getting-started/01-installation.md` per AGENTS.md — do not assume Next.js API shapes.

### `e2e/fixtures/auth.ts` — Reusable auth session

Playwright's `storageState` saves authenticated cookies so tests don't repeat login for every spec.

```typescript
import { test as base, expect } from "@playwright/test";
import type { Page } from "@playwright/test";

// Path where authenticated session cookies will be stored
export const AUTH_FILE = "e2e/.auth/user.json";

export const test = base.extend<{ authenticatedPage: Page }>({
  authenticatedPage: async ({ page }, use) => {
    // Load stored auth state
    await page.context().addCookies(
      require(AUTH_FILE).cookies
    );
    await use(page);
  },
});

export { expect };
```

### `e2e/setup/global-setup.ts` — One-time auth setup

```typescript
import { chromium } from "@playwright/test";
import path from "path";
import fs from "fs";

const AUTH_FILE = path.join(__dirname, "../.auth/user.json");

export default async function globalSetup() {
  // Skip if auth state already saved and fresh enough (<1h old)
  if (fs.existsSync(AUTH_FILE)) {
    const stat = fs.statSync(AUTH_FILE);
    const ageMs = Date.now() - stat.mtimeMs;
    if (ageMs < 60 * 60 * 1000) {
      console.log("[setup] Reusing cached auth state");
      return;
    }
  }

  const browser = await chromium.launch({ headless: false }); // headed for OAuth
  const page = await browser.newPage();

  // Navigate to app — should redirect to /login
  await page.goto(process.env.BASE_URL + "/login");
  await page.getByRole("button", { name: /continue with google/i }).click();

  // Google OAuth — requires manual login or saved credentials
  // For CI: use a service account with pre-authorized OAuth token stored in env
  await page.waitForURL("**/dashboard", { timeout: 120_000 });

  fs.mkdirSync(path.dirname(AUTH_FILE), { recursive: true });
  await page.context().storageState({ path: AUTH_FILE });
  await browser.close();
}
```
